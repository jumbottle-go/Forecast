────────────────────────────────────────────────────────────────
DATA MODELS
────────────────────────────────────────────────────────────────

NewsItem.swift defines four types:

  AIAnalysis
    summary: String
    confidencePercent: Int
    prosLabel: String       # label for pros block (e.g. "Hold", "YES")
    consLabel: String       # label for cons block
    pros: [String]          # supporting bullet points
    cons: [String]          # opposing bullet points

  Category (enum, CaseIterable, Identifiable)
    cases: forYou, finance, tech, politics, sports, science
    .symbol → SF Symbol name (e.g. "chart.line.uptrend.xyaxis" for finance)

  VoteOption (struct, Identifiable)
    id, iconName, text, subtitle, percent

  NewsItem (struct, Identifiable, Hashable by id)
    id, title, subtitle, imageURL
    source, timeAgo, votesCount
    category: Category
    isBreaking: Bool (default false)
    question: String
    options: [VoteOption]
    aiShortAnswer: String?
    aiAnalysis: AIAnalysis?

Article.swift defines three types:

  KeyFact (struct, Identifiable)
    emoji: String (SF Symbol name), text: String

  BodyElement (enum, Identifiable, Hashable)
    .text(UUID, String)
    .image(UUID, url, caption)

  Article (struct, Identifiable, Hashable by id)
    id, newsItem, keyFacts: [KeyFact], bodyParagraphs: [BodyElement]

FlashCard (struct, Identifiable — defined in MockData.swift)
  id: UUID          # same UUID as its NewsItem — required for vote tracking
  question, category, symbol, imageURL
  votesCount, aiShortAnswer, aiAnalysis

MOCK DATA (MockData.swift)
────────────────────────────────────────────────────────────────

Static data only, no backend, no persistence.

Content:
  breakingHero      1 breaking item (US-China trade war, politics)
  financeNews       3 items (Fed rates, Bitcoin $128k, Nvidia $42B quarter)
  techNews          3 items (GPT-5, Apple Vision Pro 2, Google Willow quantum)
  politicsNews      3 items (EU defense fund, NATO expansion, UN AI treaty)
  sportsNews        3 items (UCL Real vs Arsenal, F1 2026, NBA LeBron to Warriors)
  scienceNews       3 items (Mars water, mRNA cancer vaccine, AMOC collapse)
  flashNews         5 items (S&P today, Arsenal tonight, BTC $130k, Vision Pro, EUR/USD)

allNews = financeNews + techNews + politicsNews + sportsNews + scienceNews + flashNews

articles: [UUID: Article]
  Full articles for: breakingHero, financeNews[0..2], techNews[0], sportsNews[0], scienceNews[0]
  Remaining items use defaultKeyFacts / defaultBody helpers (auto-generated from newsItem fields)

flashCards: [FlashCard]
  Computed from flashNews. Each FlashCard.id == its NewsItem.id.
  This UUID parity is critical: FeedViewModel.cardVotes uses the same id for both.

Flash card YES/NO logic:
  prosLabel == "YES" → YES gets confidencePercent, NO gets remainder
  prosLabel == "NO"  → NO gets confidencePercent, YES gets remainder
  flashVoteOptions() in FlashHeroView builds the [VoteOption] pair at runtime

────────────────────────────────────────────────────────────────
COMPONENT: FlashHeroView
────────────────────────────────────────────────────────────────

Tinder-style swiping deck of FlashCards.
Visible stack: up to 3 cards (Z-stack with scale + Y-offset depth effect).
Top card: draggable. Others: static.

Card depth effect per card in stack:
  scale: 1.0 - depth * 0.035, anchor .top
  Y offset: depth * 14pt
  zIndex: idx

Drag gesture (simultaneousGesture, minimumDistance 20):
  Only activates for horizontal drags (|dx| > |dy| * 0.6)
  Live: dragOffset tracks finger
  Live: rotation = dragOffset.width / 28 degrees
  Live: swipe tint color (success/danger) up to 0.20 opacity at 160pt
  Live: YES/NO stamps fade in after ±20pt (full opacity at ±90pt)
  Release > 100pt threshold → programmatic swipe
  Release < threshold → spring back

Programmatic swipe:
  0.22s easeOut flies card off screen
  0.22s delay → remove from array + spring reset offset

Swipe directions:
  Left swipe  → YES (first option / prosLabel side)
  Right swipe → NO  (second option)
  YES button  → left swipe
  NO button   → right swipe

ArticleView integration:
  .onReceive(.articleDismissedSwipe) → identifies voted option from feedViewModel
  → triggers programmatic swipe in correct direction

Empty state (all cards swiped):
  Streak widget: "🔥 3 days in a row" + "×1.5 multiplier active"
  5 day indicators (Mo–Fr) with checkmark.circle.fill / circle icons
  Animated transition (.spring)

Card layout (cardHeight = screen height * 0.55):
  Background: AsyncImage (scaledToFill, clipped) or category gradient fallback
  Dark vignette: clear→black(0.85) gradient top→bottom
  Top bar: CategoryBadge (left) + AIInsightButton (right)
  Bottom: question headline (title2.bold, lineLimit 4)
  Button row: YES + NO in HStack (success/danger colors, 14pt radius)
  Ghost row: "Read article" button (white.opacity(0.55), outlined)

Sheet: AnalyticsSheetView with inline voting block
  On vote in sheet → swipes flash card in corresponding direction

────────────────────────────────────────────────────────────────
VOTE FLOW SUMMARY
────────────────────────────────────────────────────────────────

Hero card vote:
  VoteButton tap → FeedViewModel.castHeroVote(option)
  → heroVotedOptionId set, animation shows percent bars
  → 0.6s later: showShareSheet = true → ShareSheetView

Feed card vote:
  VoteButton tap OR horizontal swipe (100pt threshold)
  → FeedViewModel.castVote(for: id, option:) with animation
  → Card stays in place, percent bars animate in
  → Tap on voted card → NavigationLink to ArticleView

Flash card vote (FlashHeroView path A — buttons):
  YES button → programmatic left swipe → card animates off → removed
  NO button  → programmatic right swipe → card animates off → removed

Flash card vote (FlashHeroView path B — drag):
  Drag > 100pt → swipe(to:) → card flies off → removed
  Direction: left = YES (prosLabel side), right = NO

Flash card vote (FlashHeroView path C — via article):
  "Read article" → ArticleView pushes on NavigationStack
  User votes in sticky bar → castVote → 1.5s delay → dismiss()
  → 0.6s later: .articleDismissedSwipe notification
  → FlashHeroView.onReceive: looks up votedOptionId, determines direction, swipes

Flash card vote (FlashHeroView path D — AnalyticsSheetView):
  AI button → AnalyticsSheetView with VotingBlockView
  On vote: castVote → 1.5s delay → dismiss sheet → swipe in direction

ANIMATION CONVENTIONS
────────────────────────────────────────────────────────────────

All state changes: withAnimation (no direct mutations that affect layout)
Card swipe out:     .easeOut(duration: 0.22)
Card spring back:   .spring(response: 0.4, dampingFraction: 0.7)
Card removal reset: .spring(response: 0.35, dampingFraction: 0.8)
Empty/full toggle:  .spring(response: 0.45, dampingFraction: 0.78)
Vote bar fill:      .easeOut(duration: 0.6) — VoteButton internal
Onboarding lines:   .easeOut(duration: 0.4) — staggered 0.5s apart
Onboarding grid:    .spring(response: 0.5, dampingFraction: 0.8) at t=1.5s
Breaking pulse:     .easeInOut(duration: 0.9).repeatForever(autoreverses: true)
League dot pulse:   .easeInOut(duration: 1.5).repeatForever(autoreverses: true)
Wiggle hint:        .easeInOut(0.15) × 4 repeats, 1.0s delay

────────────────────────────────────────────────────────────────
KEY RULES & INVARIANTS
────────────────────────────────────────────────────────────────

1. ShareSheet triggers ONLY on hero card vote (HeroCardView), never on feed or flash cards
2. Feed cards stay in place after vote — no collapse, no scroll change
3. FlashCard.id must equal its source NewsItem.id — required for vote state sharing
4. Flash card swipe direction: left = YES (prosLabel) side, right = NO
5. "YES"/"NO" prosLabel check in flashVoteOptions must match flash NewsItem option texts
6. All UI language: English
7. Dark theme enforced everywhere (.preferredColorScheme(.dark))
8. SF Symbols only for icons (no emoji glyphs in UI elements)
9. All category filtering uses .forYou to mean "show all"
10. ArticleView fires .articleDismissedSwipe 0.6s AFTER dismiss() — not before
11. hasWiggledThisSession is a static var on FeedCardView (resets on app restart, once per session)
